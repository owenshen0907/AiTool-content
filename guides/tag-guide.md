# 标签规则

标签的作用不是做严格分类，也不是专栏归档。

标签负责横向关联，`series` 负责长期专栏。

标签的作用是：

- 帮 `/notes` 顶部快捷视图过滤
- 帮文章之间形成横向关联
- 帮你以后回看时更快找到内容

## 1. 快捷视图对应标签

`/notes` 顶部固定 5 个快捷视图，本质上只是标签匹配：

| 视图 | 命中标签 |
| --- | --- |
| 产品 | `产品` / `product` |
| 生活 | `生活` / `life` / `daily` |
| AI | `AI` / `ai` |
| 工具 | `工具` / `tools` / `software` |
| 全部 | 不过滤 |

只要一篇文章包含这些标签之一，就会进入对应视图。

## 2. 推荐标签写法

推荐一篇文章同时有两类标签：

- 一级标签：让它进入快捷视图
- 二级标签：描述这篇文章的具体主题

例如：

```yaml
tags: [产品, AI, 配音实验, 日语学习]
```

或者：

```yaml
tags: [daily, 生活, 东京, 工作节奏]
```

## 3. 常用标签建议

可参考：

- `产品`
- `AI`
- `工具`
- `生活`
- `daily`
- `日语学习`
- `配音`
- `视频`
- `实验`
- `写作`
- `运营`
- `思考`

## 4. 不建议的做法

- 一篇文章只打一个非常抽象的标签
- 把标签写成长句
- 同一意思混用过多版本

例如：

- `产品方向判断和接下来一个季度路线`
- `工具工具`
- 同时写 `AI` / `ai` / `人工智能` / `大模型` 但没有理由

## 5. 简单原则

如果不知道怎么打标签，就按这个顺序：

1. 它应该进入哪个快捷视图
2. 它在讲哪一个具体主题
3. 如果它属于长期系列，把系列名称写到 `series` 字段，而不是只依赖标签

这样一般就够用了。

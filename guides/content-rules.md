# 内容规则

这份文档定义 `AiTool-content` 里文章内容的基础约束。

## 1. 内容目标

`/notes` 不是 CMS，也不是纯时间线碎碎念。

它更像：

- 公开工作记录
- 产品判断文档
- 实验与过程笔记
- 个人长期写作入口

所以每篇文章都应该至少满足一条：

- 有一个明确判断
- 有一个具体过程
- 有一个阶段结果
- 有一个值得留档的素材集合

## 2. Frontmatter 规范

每篇文章必须有 frontmatter：

```yaml
---
title: 标题
date: 2026-05-02
tags: [产品, AI, daily]
cover: https://cdn.example.com/cover.jpg
excerpt: 一句话摘要
draft: false
---
```

字段规则：

- `title`：必填
- `date`：必填，格式必须是 `YYYY-MM-DD`
- `tags`：推荐填写，支持自由标签
- `cover`：可选，作为列表封面
- `excerpt`：可选，列表摘要；不写时网站取正文首段
- `draft`：可选，`true` 时网站不展示

## 3. 文件命名

推荐格式：

```text
YYYY-MM-DD-slug-name.md
```

例如：

- `2026-05-02-new-direction.md`
- `2026-05-02-api-lab-update.md`

说明：

- 文件名就是 URL slug
- 日期推荐加在文件名前面，方便仓库里排序
- 真正用于站点排序的是 frontmatter 里的 `date`

## 4. 正文结构建议

建议用这个顺序：

1. 一句开场判断
2. 发生了什么
3. 你怎么理解这件事
4. 当前结果 / 阶段结论
5. 下一步

不要求每篇都很长，但最好别只写一句流水账。

## 5. 写作风格

推荐：

- 句子直接
- 段落短
- 每一节只讲一个判断
- 尽量让读者知道“你为什么这样做”

不推荐：

- 一大段没有分节的长文
- 只有事实，没有判断
- 只有情绪，没有具体内容
- 为了“像文章”而写很多空话

## 6. 发布标准

一篇文章满足下面几条，就可以发：

- 标题清楚
- 日期正确
- 标签合理
- 开头能让人知道这篇在讲什么
- 如果有图/音/视频，能正常加载
- 结尾至少有一句结论或下一步
